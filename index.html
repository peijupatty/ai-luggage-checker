🔴 這些必帶，沒得商量

      🟢 這些到當地買就好

        這裡公開系統實際送給 Gemini 的完整提示詞（Prompt）。設計上用了四個 Prompt Engineering 技巧：① 角色設定——賦予 AI「行李檢查員」的人格與口語語氣；② 條件注入——把你的行程、身體需求、黑歷史動態填入提示詞；③ 格式約束——強制只輸出指定結構的 JSON，讓程式能穩定解析；④ 負面指令——明確列出既有清單，禁止 AI 推薦重複物品。

        
        
        
          
        最近一次「智慧分類」的 Prompt：單一分類任務用短提示就夠——任務越聚焦，提示越短、速度越快、成本越低。
        
          
        
          
        
        
        
        
        
          
        最近一次「拍照核對」的 Prompt：多模態任務——文字提示與圖片一起送出，並把清單名稱注入提示中要求 AI「逐字比對」，這是讓影像辨識結果能對接程式邏輯的關鍵。

        系統設定
        
      
        ⚙️ AI 連線設定
        
      
        貼上 Google AI Studio 的 Gemini API 金鑰，分析摘要、反問和行李建議就會升級成「真 AI 即時生成」。
        
    
        
    
        
      
        🔑 Gemini API 金鑰
        
      
        
        
        
        
        
      
        
      
        尚未設定金鑰。沒有金鑰也能正常使用，系統會以「📦 離線模擬模式」運作。
        
      
        
        運作方式：按下「產生打包清單」時，清單與風險分析永遠由本機規則引擎產生（保證不漏掉護照這種關鍵物品）；AI 分析摘要、三個反問、行李建議則會即時呼叫 Gemini 生成，真正認識你填的目的地。AI 連線失敗或逾時會自動退回離線模擬，demo 不會開天窗。

        
        金鑰安全：金鑰只存在「你這台電腦的瀏覽器」裡，不會寫進這個 HTML 檔案，檔案可以放心分享。但請不要把金鑰本身傳給別人或截圖進投影片。

        
        取得金鑰：到 aistudio.google.com → Get API key，免費額度對 demo 綽綽有餘。

        🎯 系統簡介
        
        
        本系統幫助使用者在旅行前產生個人化行李清單，並透過勾選檢查、漏帶風險分析、出門前緊急檢查與回程遺漏檢查，降低忘記帶重要物品的機率。系統採用「混合式 AI 架構」：邏輯明確、不容出錯的部分（清單產生、風險分級）由本機規則引擎負責；需要語言與認知能力的部分（分析摘要、行李建議、物品分類、影像辨識）則串接 Google Gemini 即時生成，並設計連線失敗時的自動降級機制。
        
      
        
    
        

    
        
      
        
        
        🧠 系統運用的 AI 概念
        
        
        
          
        生成式 AI
        個人化分析與建議
        將使用者行程組成 Prompt，由 LLM 即時生成分析摘要、反問與行李建議，真正「認識」目的地。
        
          
        文本分類
        智慧新增品項
        輸入物品名，AI 自動判斷所屬分類與漏帶風險等級，是典型的 classification 任務。
        
          
        電腦視覺・多模態
        拍照核對行李
        上傳行李照片，AI 辨識物品並自動勾選清單、提醒未見到的高風險物品。
        
          
        Prompt Engineering
        提示詞工程
        運用角色設定、條件注入、JSON 格式約束、負面指令四項技巧，並以「透明窗」公開展示。
        
          
        規則式系統
        降級容錯
        AI 失敗時，分類與分析自動切換至本機關鍵字規則引擎，保證系統永遠可用。
        
          
        資料飛輪
        閉環學習
        旅程成績單回收忘帶結果，寫入黑歷史並於下次加權，系統越用越懂使用者。
        
        
        
      
        
    
        

    
        
      
        
        
        🏗️ 混合式架構：AI ＋ 規則引擎
        
        
        按下「產生清單」時的資料流：
        
        
        
          使用者設定→
          規則引擎產生基礎清單→
          並行呼叫 Gemini
        
        
        
        
          ✅ 成功 → AI 摘要／建議／加碼品項
        
        
        
        
          ⚠️ 失敗 → 自動降級為離線句庫
        
        
        
        設計理由：護照、藥品這類「絕對不能漏」的關鍵物品交由規則引擎可保證 100% 不遺漏；而「像人一樣說話」的部分交給 AI。兩者各司其職，並透過降級機制確保即使斷網或 API 額度用盡，系統核心功能仍完整可用。

        ⚙️ 使用技術
        
        
        前端：HTML / CSS / JavaScript（單一檔案、無後端）。AI：Google Gemini API（多模型自動輪替、JSON 結構化輸出、多模態圖片輸入）。資料持久化：瀏覽器 localStorage。設計：RWD 響應式、卡片式介面、多步驟導覽與 hash 路由。
